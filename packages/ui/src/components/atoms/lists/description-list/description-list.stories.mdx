# Description List

A description list is commonly used to implement a glossary or to display metadata. It represents a list of key-value pairs.

## Contents

A description list is composed of `<Term />` and `<Description />` components.

### Single term and description

You can use a single term with a single description:

<Canvas>
  <Story name="Contents: Single term & description">
    <DescriptionList>
      <Term>A term</Term>
      <Description>A description of the term.</Description>
    </DescriptionList>
  </Story>
</Canvas>

### Multiple terms, single description

You can also use a multiple terms with a single description:

<Canvas>
  <Story name="Contents: Multiple terms, single description">
    <DescriptionList>
      <Term>A term</Term>
      <Term>A synonym</Term>
      <Term>Another synonym</Term>
      <Description>A description of the term.</Description>
    </DescriptionList>
  </Story>
</Canvas>

### Single term, multiple descriptions

You can also use a single term with multiple descriptions:

<Canvas>
  <Story name="Contents: Single term, multiple descriptions">
    <DescriptionList>
      <Term>A term</Term>
      <Description>A description of the term.</Description>
      <Description>Another description of the term.</Description>
    </DescriptionList>
  </Story>
</Canvas>

### Multiple terms and descriptions

You can also combine the examples above to define multiple terms with multiple corresponding descriptions.

<Canvas>
  <Story name="Contents: Multiple terms & descriptions">
    <DescriptionList>
      <Term>A first term</Term>
      <Description>A description of the first term.</Description>
      <Term>A second term</Term>
      <Term>A synonym</Term>
      <Term>Another synonym</Term>
      <Description>A description of the second term.</Description>
      <Term>A third term</Term>
      <Description>A description of the third term.</Description>
      <Description>Another description of the third term.</Description>
    </DescriptionList>
  </Story>
</Canvas>

### Group of terms and descriptions

You can also use groups to wrapper each term(s)/description(s) association. They can be use for styling purposes or to improve code readability when mixing a lot of terms and descriptions.

Here an example with the above example:

<Canvas>
  <Story name="Contents: Groups of terms & descriptions">
    <DescriptionList>
      <Group>
        <Term>A first term</Term>
        <Description>A description of the first term.</Description>
      </Group>
      <Group>
        <Term>A second term</Term>
        <Term>A synonym</Term>
        <Term>Another synonym</Term>
        <Description>A description of the second term.</Description>
      </Group>
      <Group>
        <Term>A third term</Term>
        <Description>A description of the third term.</Description>
        <Description>Another description of the third term.</Description>
      </Group>
    </DescriptionList>
  </Story>
</Canvas>
